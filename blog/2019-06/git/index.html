git

  • 新建一个用于建立仓库的空文件夹mkdir new
  • cd进入新建的文件夹，执行git init初始化仓库，产生.git文件夹，隐藏文件夹可用ls -a查看
  • 在仓库内，对于新建或改动的文件可用git add name添加到缓存区，再使用git commit -m “备注”提交到仓库管理，git add name可添加多个文件。
  • 使用git status可以查看仓库当前的状态，使用git diff name可以查看相关文件的改动。
  • 使用git log可查看提交的历史记录，git log --pretty=oneline可简单输出信息。
  • 使用git reset --hard HEAD^可回退到上一版本。可通过git reflog找到回退的版本号，使用git reset --hard commitid回到相应版本。
  • 当想丢弃工作区的修改时，用命令git checkout -- file取消工作区修改。当改乱了工作区某个文件，还添加到了暂存区时，想丢弃修改，先用命令git reset HEAD <file>丢弃暂存区内容，再使用git checkout -- file丢弃工作区。当提交修改到版本库，可用git reset --hard commitid回退到相应版本。
  • 使用git rm file和git commit -m “备注”可删除仓库的文件，同时本地文件也将被删除，可用`git reset –hard commitid恢复工作区和仓库的内容。

github

  • 在用户主目录下，看看有没有.ssh目录，如果有，再看看这个目录下有没有id_rsa和id_rsa.pub这两个文件。如果没有，使用ssh-keygen -t rsa -C "youremail@example.com"后三次回车，复制id_rsa.pub内容在github中SSH and GPG keys填入。
  • 在github新建仓库，选择本地文件夹，使用git remote add origin 仓库链接将本地仓库于远端仓库关联。当远端未初始化想把本地推向远端时，使用git push -u origin master之后只需要git push origin master。当远端已经初始化并想把克隆下来，使用git clone 地址或使用git initandgit remote add origin 仓库链接andgit pull origin master。
  • 使用git remote -v可显示远端仓库详细的信息

分支管理

  • 使用git checkout -b 分支创建并切换分支，使用git checkout 分支可切换分支。用git branch命令查看当前分支，当前分支前面会标一个*号。git merge 分支用于合并指定分支到当前分支，然后可用git branch -d 分支删除分支。

  • 用带参数的git log --graph --pretty=oneline --abbrev-commit也可以看到分支的合并情况。

  • Fast forward模式在合并后会删除分支后，会丢掉分支信息。可使用git merge --no-ff -m "merge with no-ff" dev禁用Fast forward。

  • 分支管理：首先，master分支应该是非常稳定的，也就是仅用来发布新版本，平时不能在上面干活；干活都在dev分支上，也就是说，dev分支是不稳定的，到某个时候，比如1.0版本发布时，再把dev分支合并到master上，在master分支发布1.0版本；每个人都在dev分支上干活，每个人都有自己的分支，时不时地往dev分支上合并就可以了。

  • 使用git stash功能可以把当前工作现场“储藏”起来，用git stash list查看。可使用git stash apply stash@{n}恢复，但是恢复后，stash内容并不删除，你需要用git stash drop来删除；也可以用git stash pop，恢复同时把stash内容也删了。

  • 使用git branch -D 分支可强行删除还未提交的分支。
